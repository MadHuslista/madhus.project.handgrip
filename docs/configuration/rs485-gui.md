# RS485 GUI Configuration Reference

**Status:** Canonical root configuration reference  
**Component:** `RS485_GUI`  
**Detailed component doc:** `RS485_GUI/docs/configuration.md`  
**Config source:** `RS485_GUI/config/config.yaml`

## Summary

`RS485_GUI/config/config.yaml` controls the reference acquisition board connection, GUI behavior, file logging, and ZeroMQ IPC publication to `LSL_Bridge`. Display throttling is not acquisition downsampling.

## Configuration table

| Key                                  | Type        | Default                      | Allowed range / values                              | Operational impact                             | When to change                                | Failure risk                                                 |
| ------------------------------------ | ----------- | ---------------------------- | --------------------------------------------------- | ---------------------------------------------- | --------------------------------------------- | ------------------------------------------------------------ |
| `session.session_id`                 | string/null | `null`                       | Any stable session label.                           | Propagates session identity to logs/IPC.       | During calibration or structured experiments. | Logs hard to associate with sessions.                        |
| `app.log_level`                      | string      | `INFO`                       | `DEBUG`, `INFO`, `WARNING`, `ERROR`.                | App-level diagnostics.                         | Debug parser/transport/UI issues.             | Too verbose or insufficient diagnostics.                     |
| `ui.port`                            | int         | `8088`                       | Free TCP port.                                      | Browser UI port.                               | Port conflict.                                | App fails to start or binds wrong port.                      |
| `ui.refresh_interval_s`              | float       | `0.1`                        | Positive seconds.                                   | UI refresh cadence.                            | Tune browser load/responsiveness.             | Browser lag if too low; sluggish UI if too high.             |
| `ui.max_render_plot_points`          | int         | `700`                        | Positive integer.                                   | Limits points sent to browser plots.           | Slow browser rendering.                       | Misunderstood as data downsampling if not documented.        |
| `ui.display_max_samples_per_second`  | int         | `30`                         | `0` disables cap; positive display cap.             | Display-only sample rate cap.                  | Reduce UI load.                               | Can hide fast events visually, but not in logs/IPC.          |
| `logger.enabled`                     | bool        | `true`                       | `true`/`false`.                                     | Enables file logging.                          | Disable only for test/demo.                   | Loss of provenance if disabled during experiments.           |
| `logger.directory`                   | path        | `RS485_GUI/logs`             | Writable path.                                      | Log output root.                               | Organize sessions.                            | Missing logs or overwritten logs.                            |
| `logger.write_mode`                  | string      | `overwrite`                  | `overwrite` or supported append mode.               | Controls whether old logs are replaced.        | Preserve multiple sessions.                   | Accidental data loss if overwrite surprises operator.        |
| `logger.raw_signal_filename`         | string      | `raw_signal.ndjson`          | Filename.                                           | Raw frame log.                                 | Rarely change.                                | Tools/docs may not find raw log.                             |
| `logger.interpreted_signal_filename` | string      | `interpreted_signal.ndjson`  | Filename.                                           | Parsed measurement log.                        | Rarely change.                                | Loss of interpreted traceability.                            |
| `logger.gui_signal_filename`         | string      | `gui_signal.csv`             | Filename.                                           | GUI-visible signal export.                     | CSV workflow changes.                         | Viewer/replay examples may break.                            |
| `ipc.enabled`                        | bool        | `true`                       | `true`/`false`.                                     | Enables ZMQ publication.                       | Reference-only debugging or offline UI use.   | `LSL_Bridge` cannot publish `HandgripReference` if disabled. |
| `ipc.bind`                           | string      | `tcp://127.0.0.1:5557`       | Valid ZMQ PUB bind endpoint.                        | IPC endpoint for bridge.                       | Multi-host or port conflict.                  | Bridge cannot connect.                                       |
| `ipc.topic`                          | string      | `rs485.measurement.v1`       | Must match bridge `rs485_ipc.topic`.                | Measurement topic contract.                    | Only with coordinated bridge migration.       | Reference stream missing if mismatch.                        |
| `ipc.event_topic`                    | string      | `rs485.event.v1`             | Event topic string.                                 | Optional event diagnostics.                    | Add event consumers.                          | Diagnostics missing if mismatched.                           |
| `ipc.signal_key`                     | string      | `net_value`                  | Valid parsed measurement key.                       | Selects reference value sent as primary force. | If board/parser field changes.                | Wrong reference signal used by bridge.                       |
| `serial.default_port`                | string      | empty                        | `/dev/ttyUSB*`, `/dev/serial/by-id/*`, `COMx`, etc. | Selects USB-RS485 adapter.                     | Every bench/host setup.                       | No data or wrong device.                                     |
| `serial.default_baudrate`            | int         | `460800`                     | Board-supported baud.                               | RS485 transport speed.                         | Must match board `501.br`.                    | No frames/garbled frames if mismatch.                        |
| `device.mode`                        | string      | `active_send`                | `active_send`, `modbus_rtu` where supported.        | Selects acquisition method.                    | Fallback to Modbus if Active-Send unstable.   | Wrong parser/transport behavior.                             |
| `device.slave_address`               | int         | `1`                          | Board Modbus address.                               | Modbus/Active-Send frame source.               | If board address changes.                     | Poll/read mismatch.                                          |
| `device.active_send_frequency_code`  | int         | `8`                          | Board-specific code; expected 500 Hz profile.       | Controls Active-Send output rate.              | Tune reference rate.                          | Transport overload or insufficient reference timing detail.  |
| `active_send.default_parser_profile` | string      | `modbus_rtu_response_11regs` | Supported parser profile.                           | Parses board active-send payload.              | If firmware/board frame shape changes.        | Malformed frames or wrong values.                            |
| `active_send.timestamp_policy`       | string      | `batch_end_anchored`         | Supported timestamp policy.                         | Assigns timing to received frames.             | Timing diagnostics/latency tuning.            | Misleading timing if wrong.                                  |
| `logging.module_levels`              | map         | module-specific              | Logger names to levels.                             | Fine-grained diagnostics.                      | Debug a subsystem.                            | Noise or missing logs.                                       |
